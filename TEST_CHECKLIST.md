# 测试清单

用于前后端联调、自测和交付前快速验收。

## 一、启动检查

- 微信开发者工具可以正常打开 `d:\1work\MBTI`
- 小程序编译通过，无首屏阻断错误
- 后端服务已启动在 `http://127.0.0.1:3000`
- [app.js](/d:/1work/MBTI/app.js:1) 中 `useMock` 与 `apiBaseUrl` 配置符合当前测试目标

## 二、mock 模式检查

- `useMock = true` 时首页可正常打开
- 不启动后端时，首页、答题页、报告页、历史页仍能完整走通
- mock 模式下不会发往真实 `apiBaseUrl`

## 三、真实接口主流程

- 首页能成功拉到测评配置
- 首页文案正确使用配置字段：
  - `themeTitle`
  - `startCopy`
- 20 题可以逐题选择和切换
- 漏答时不允许提交
- 提交成功后能拿到 `recordId`
- 报告页先进入 `pending`
- 报告页随后进入 `ready`
- 报告页正确展示：
  - `themeTitle`
  - `reportTitle`
  - `mbtiType`
  - 4 个固定模块
  - 每模块 3 条内容
- 报告页显示 `endCopy`
- 结果页分享入口可点击
- 历史页能看到刚生成的记录
- 历史页点击记录后可再次打开报告详情

## 四、结果校验

- 全选 `A` 时，返回 `ESTJ`
- 全选 `B` 时，返回 `INFP`
- 再次打开已成功报告时，能直接读取已有结果
- `failed` 结果不会进入历史列表展示

## 五、异常态检查

- 配置拉取失败时，首页有明确反馈
- 提交接口失败时，答题页有明确反馈
- `GET /report/{recordId}` 返回 `failed` 时，报告页只展示失败态，不显示半成品内容
- 缺少 `recordId` 打开报告页时，不会卡死在空白页
- 网络异常或接口超时后，页面仍有可恢复路径

## 六、微信小程序专项检查

- 开发者工具中可通过 `Network` 看到真实请求
- 真机联调时，已将 `apiBaseUrl` 改为局域网 IP 或可访问的 `https` 域名
- 正式环境前已确认合法域名配置

## 七、交付前确认

- 前端已明确区分 mock 演示模式和真实接口模式
- 后端接口与 [MBTI_backend_api_v1.md](/d:/1work/MBTI/MBTI_backend_api_v1.md:1) 保持一致
- README 已说明启动方式、联调方式和配置项用途
